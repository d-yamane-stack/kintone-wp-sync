## 1. 見出しのルール

### 1-1. H1（記事タイトル）

- **クラス名**: `heading1 mt-0 mb-3`
- **役割**: 記事の主タイトル。ページ内に1つのみ。
- **文字数の傾向**: 30〜50文字程度
- **書き方パターン**:
  - 疑問形・感嘆符（！）を積極的に使う
  - 「」（鍵括弧）でキーワードを強調する
  - サブタイトルを「：」で繋げることもある

**実例:**
```
「築10年・20年・30年」の節目でやるべきリフォーム優先順位ガイド：後悔しない予算配分のコツ
補助金を賢く使う！2026年度版「住宅リフォーム支援」の最新情報
収納不足を解決！「クローゼットの整理」と「棚の増設」で衣替えを楽に
```

### 1-2. H2（セクション見出し）

- **クラス名（本文セクション）**: `is-style-heading`
- **クラス名（まとめ）**: クラスなし（素のH2）
- **クラス名（著者プロフィール）**: `column-profile-title`
- **役割**: 本文を3〜4つのメインセクションに分ける
- **数字プレフィックス**: 必ず `1 ` `2 ` `3 ` の数字を見出し冒頭に付ける（目次アンカーと連動）
- **文字数の傾向**: 20〜35文字程度
- **書き方パターン**:
  - 数字 + スペース + 内容という形式（例: `1 窓や断熱に特化した「先進的窓リノベ」等の継続`）
  - 「」（鍵括弧）でキーワードを囲む
  - 末尾に「まとめ」という固定のH2を配置

**実例:**
```html
<h2 class="is-style-heading">1 窓や断熱に特化した「先進的窓リノベ」等の継続</h2>
<h2 class="is-style-heading">2 子育て世帯・若者夫婦への手厚い優遇策</h2>
<h2 class="is-style-heading">3 予算切れに注意！早めの計画と「登録事業者」選び</h2>
<h2>まとめ</h2>
```

### 1-3. H3（サブセクション・関連コンテンツ）

- **クラス名（おすすめコラム）**: `heading1 my-0`
- **クラス名（関連記事カード）**: `card1-heading1`
- **クラス名（著者専門領域）**: `column-profile-expertise-title`
- **役割**: 本文内のH2セクション内には基本的にH3は使われない。H3は主に記事末尾の「おすすめ関連コラム」見出しと、関連記事カードのタイトルに使われる。
- **書き方パターン**: 関連記事のタイトルをそのまま使用

**実例:**
```html
<h3 class="heading1 my-0">このコラムを見た人におすすめ</h3>
<h3 class="card1-heading1">トイレ増設費用、工期の実例紹介｜補助金、簡単な増設方法、注意点なども解説</h3>
```

### 1-4. H4（詳細情報）

- **クラス名**: `column-profile-expertise-heading1`
- **役割**: 著者プロフィールセクション内の詳細項目（「免許登録」「資格情報」など）のみに使用。本文記事内では使われない。

### 見出し階層まとめ

```
H1（heading1 mt-0 mb-3）       ← 記事タイトル（1つのみ）
├── H2（is-style-heading）      ← 本文セクション（数字付き、3〜4個）
│   └── ※H3は本文内では未使用
├── H2（クラスなし）            ← まとめ
├── H3（heading1 my-0）         ← このコラムを見た人におすすめ
│   └── H3（card1-heading1）    ← 関連記事タイトル
└── H2（column-profile-title）  ← 著者プロフィール
    ├── H3（column-profile-expertise-title）
    └── H4（column-profile-expertise-heading1）
```

## 2. 画像挿入のルール

### 2-1. 記事メインビジュアル（アイキャッチ）

- **挿入位置**: H1タイトルの直後、記事本文の冒頭（目次より前）
- **コンテナ**: `<figure class="wp-block-image size-large">`
- **サイズ**: 1024×705px（アスペクト比約1.45:1）
- **altテキスト**: **空文字（alt=""）**が全記事共通。装飾的画像として扱っている。
- **クラス名**: `wp-image-{ID} lazyloaded`（遅延読み込み対応）
- **画像パス**: `/refresh2022/wp-content/uploads/{年}/{月}/{ハッシュ値}-1024x705.{拡張子}`

**実例:**
```html
<figure class="wp-block-image size-large">
  <img 
    src="https://jube.co.jp/refresh2022/wp-content/uploads/2026/04/8c8644ea6c7bd571f18e7c555a0fd2ad-1024x705.jpg"
    data-src="https://jube.co.jp/refresh2022/wp-content/uploads/2026/04/8c8644ea6c7bd571f18e7c555a0fd2ad-1024x705.jpg"
    alt=""
    class="wp-image-123233 lazyloaded"
    width="1024"
    height="705"
  >
</figure>
```

### 2-2. セクション内画像

- **挿入位置**: 現在分析した3記事では、本文セクション内にオリジナル画像は1枚のみ（メインビジュアルのみ）
- **プレースホルダー**: 各H2セクション内にSVGプレースホルダーが配置されている（`data:image/svg+xml` のlazyload画像）。これは記事によっては実画像が入る可能性がある構造。
- **1記事あたりの実画像数**: メインビジュアル1枚 + セクション数分のプレースホルダー（3〜4枚）が基本構造

### 2-3. 関連記事カード画像

- **コンテナ**: `<figure class="card1-img">`
- **altテキスト**: 空文字
- **プレースホルダー**: `750x550.png`（固定のプレースホルダー画像）

### 2-4. altテキストのパターン

| 画像の種類 | altテキスト |
|-----------|------------|
| 記事メインビジュアル | 空文字（alt=""） |
| セクション内挿絵 | 空文字（alt=""） |
| 関連記事サムネイル | 空文字（alt=""） |
| SNSアイコン | "Instagram" など SNS名 |

**共通ルール**: コンテンツ画像は全てaltテキストが空文字。画像は装飾的・視覚補助的に扱われており、文章内容をaltで補完する設計にはなっていない。

## 3. 段落構成のルール

### 3-1. 記事全体の構成テンプレート

```
[1] 記事メタ情報
    - カテゴリタグ（例: #お金、#収納）
    - 公開日・最終更新日

[2] H1（記事タイトル）

[3] メインビジュアル（figure.wp-block-image）

[4] 導入段落（p × 2〜3）
    - 問題提起（読者の共感を引き出す）
    - 記事で解決できることの予告
    - 「今回は〜を解説します」という締め

[5] スピーチバルーン（おすすめ対象読者）
    - class: wp-block-liquid-speech-balloon liquid-speech-balloon-wrap liquid-speech-balloon-00
    - 「この記事は、次の人におすすめです！」で始まる
    - 対象読者を箇条書き（3〜4項目）

[6] 目次（div#toc_container.toc_white.no_bullets）
    - 自動生成のWordPress目次プラグイン
    - H2見出しへのアンカーリンク

[7] 本文セクション × 3（H2.is-style-heading ＋ 本文）
    各セクションの構成:
    ├── H2（数字付き）
    ├── p（本文段落 × 2〜3）
    └── ul.is-style-ul-style1（箇条書き × 3項目）

[8] まとめ（H2 + p × 1〜2）
    - H2「まとめ」（クラスなし）
    - 記事全体の要点をまとめる段落

[9] このコラムを見た人におすすめ
    - H3.heading1.my-0
    - 関連記事カード × 3件

[10] 著者プロフィール
    - H2.column-profile-title「ハウジング重兵衛 編集部のプロフィール」
```

### 3-2. 段落（p タグ）のルール

- **クラス名**: 基本的に空（クラスなし）。`toc_title` は目次専用。
- **1段落あたりの文字数**:
  - 導入部: 40〜120文字（短め〜中程度）
  - 本文セクション: 50〜200文字（中程度）
  - まとめ: 100〜200文字（やや長め）
- **改行の入れ方**: HTMLのpタグを区切りとして改行。1つのpタグ内に改行（`<br>`）は使わない。
- **書き方の傾向**:
  - 口語的・親しみやすいトーン
  - 読者への問いかけで始まるパターンあり（例：「〜していませんか」）
  - 最後のまとめ段落には「千葉・茨城で120余年」などの会社実績・CTA誘導が入る

### 3-3. リスト（ul/li）のルール

- **本文内リストのクラス名**: `is-style-ul-style1`
- **目次リストのクラス名**: `toc_list`
- **配置位置**: 各H2セクションの末尾（段落の後）に配置
- **項目数**: 各セクション1つのリストに3項目が固定パターン
- **olタグ**: 分析した記事では使用なし（ulのみ）
- **内容の傾向**: 各リスト項目は1〜2文で、そのセクションの要点・ポイントをまとめる役割

**実例:**
```html
<ul class="is-style-ul-style1">
  <li>高性能な断熱窓へのリフォームは、工事費用の相当額を補助金で補える可能性がある</li>
  <li>家全体の断熱性能を高めるリフォームは、健康維持（ヒートショック防止）にも大きく寄与する</li>
  <li>最新の「断熱基準」に適合する工事が対象となるため、製品選びの段階での確認が必須となる</li>
</ul>
```

### 3-4. スピーチバルーン（おすすめ対象読者）

- **外側コンテナ**: `wp-block-liquid-speech-balloon liquid-speech-balloon-wrap liquid-speech-balloon-00`
- **テキスト部分**: `liquid-speech-balloon-text`
- **矢印部分**: `liquid-speech-balloon-arrow`
- **配置**: 導入段落の後、目次の前
- **内容の書き方**: 「この記事は、次の人におすすめです！」の後に、対象読者を箇条書き（・記号）で列挙

**実例:**
```
この記事は、次の人におすすめです！
・断熱窓や高効率給湯器など、省エネリフォームを検討しており補助金を使いたい方
・「子育て世帯」や「若者夫婦」を対象とした、手厚い支援策の詳細を知りたい方
```

## 4. ボタン・リンクのルール

### 4-1. グローバルナビゲーション（記事本文外）

- 記事上部: 「お問い合わせ」「無料見積」「来店予約」「LINE相談」ボタン
- これらは全記事共通のレイアウト要素（本文には含まない）

### 4-2. 記事本文内のリンク

- **CTAボタン**: 本文内にWPブロックのボタンは挿入されていない（3記事共通）
- **内部リンク**: 目次アンカーリンク（`href="#1"` `href="#2"` `href="#3"` `href="#i"`）
- **外部リンク**: 分析した記事では本文中に外部リンクなし

### 4-3. 記事末尾の関連記事リンク

- **構造**: 画像（figure.card1-img）+ 日付 + H3（card1-heading1）のカードUI
- **件数**: 3件固定
- **配置**: H3「このコラムを見た人におすすめ」の直下
- **リンク先**: 同一サイト内のコラム記事

## 5. その他の特徴的な要素

### 5-1. 記事メタ情報

```
#カテゴリ名
公開日: YYYY年M月D日　最終更新日: YYYY年M月D日
```

- カテゴリタグ: `#お金`、`#収納`、`#ldk リフォーム 費用` など
- 公開日と更新日の両方を表示

### 5-2. 目次

- プラグイン: WordPress Table of Contents（`#toc_container.toc_white.no_bullets`）
- 目次タイトル: `<p class="toc_title">目次</p>`
- アンカーIDのパターン: H2の数字と内容から自動生成（例: `#1_1015` `#2_2025`）

### 5-3. 著者プロフィール

- **見出し**: `<h2 class="column-profile-title">ハウジング重兵衛 編集部のプロフィール</h2>`
- **画像コンテナ**: `<figure class="column-profile-img">`
- **専門領域**: `<h3 class="column-profile-expertise-title">リフォームを中心とした住宅業界</h3>`
- **詳細項目**: `<h4 class="column-profile-expertise-heading1">免許登録</h4>`

### 5-4. 遅延読み込み（Lazy Load）

- 画像は `lazyload` / `lazyloaded` クラスで遅延読み込み実装
- `data-src` 属性に実際のURL、`src` には `data:image/svg+xml` のプレースホルダーを使用

## 7. タイトル・見出しの書き方チェックリスト

記事を生成する際の確認事項:

- [ ] H1タイトルは30〜50文字か
- [ ] H1に「！」や「？」の感嘆符・疑問符を使っているか
- [ ] H1に「」（鍵括弧）でキーワードを強調しているか
- [ ] H2には「1 」「2 」「3 」の数字プレフィックスを付けているか
- [ ] H2（本文セクション）に `is-style-heading` クラスを付けているか
- [ ] 「まとめ」H2にはクラスを付けていないか
- [ ] 各H2セクション末尾に `is-style-ul-style1` クラスのulを3項目で配置しているか
- [ ] スピーチバルーン（対象読者）を導入部と目次の間に配置しているか
- [ ] メインビジュアルのaltテキストは空文字か
- [ ] まとめ段落に会社の実績・サービス誘導の文言が含まれているか
